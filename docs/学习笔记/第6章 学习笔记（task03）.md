第6章学习笔记

1、【Llama学习】

（1）概述：Llama2 采用 **Decoder-Only** 架构（纯 Transformer 解码器堆叠），适配自回归文本生成任务；起核心改进点可总结为预归一化（Pre-Normalization）、分组查询注意力（GQA）、SwiGLU 前馈网络、RMSNorm 归一化、RoPE 旋转位置编码、残差连接；

（2）核心改进点：

①预归一化(RMSNorm)：移除均值中心化，仅通过均方根缩放，保留可学习增益参数（γ），简化 LayerNorm提升效率；

②旋转位置编码（RoPE）：相对位置编码，通过复数乘法旋转 Query/Key 向量注入位置信息（非加法）；

③分组查询注意力（GQA）：多个 Query 头共享一组 Key/Value 头（MHA 是 1:1，MQA 是全共享，GQA 为中间态），具体通过`KV 头复制 n_rep 次（n_rep = n_heads/n_kv_heads），与 Q 头数对齐，优化多头注意力（MHA）的 KV 缓存和计算开销；

④SwiGLU 前馈网络：门控机制（三个线性层W1/W3/W2） + Swish 激活（silu），中间隐藏层维度做硬件友好型对齐，替代传统 ReLU FFN，增强非线性表达能力；

（3）工程实践启发：

①减法思维：预归一化RMSNorm去除了均值中心化，既保持了归一化的效果，又提高了计算效率；

②**计算效率的平衡**：GQA通过共享KV头减少计算量，这种设计在几乎不损失性能的前提下大幅提升了推理速度。在实际应用中，类似的**计算共享**策略值得探索。

③内存管理的艺术：KV缓存、RoPE预计算等技术都体现了对内存资源的精细管理。在大模型应用中，**内存优化**往往能带来比计算优化更显著的收益。

2、【MoE 混合专家模型】

（1）概述：

MOE通过“稀疏激活”(分治策略和动态条件计算)解决稠密模型算力成本过高的问题，打破 “性能靠堆算力” 的范式，实现了大规模参数与低推理成本的平衡，让大模型在推理的时候不用全参数参与，而是通过激活部分专家网络实现“专才干专事”的低成本高精度推理。

（2）分治思想设计：

①自适应局部专家混合（1991）：早期是“专家网络 + 门控网络” 架构的分治思想，手动设计竞争损失函数（负对数似然）做权重解耦，仅更新topk专家和门控权重；

②深度MOE(2013): 提出MoE 模块化，通过层级化门控实现专家组合路径指数级增长，实现了特征自动解耦；

③稀疏门控MOE（2017经典）：提出稀疏门控 MoE 层（带噪声稀疏门控仅激活K个，负载均衡引入辅助损失避免专家崩塌），实现 “条件计算”（输入决定专家选择），达到不增加计算量却大幅提升参数量的效果；

（3）MOE核心技术：

①专家网络：双层全连接（FFN），独立处理局部子任务；

②门控网络：动态分配 Token 到专家，输出混合权重；

③路由策略：Top-k（k=1/2/4）是主流，平衡计算效率与性能；

④负载均衡：辅助损失、专家容量、Token Dropping 是关键手段；

（4）工程实践启发：

①并行策略：非 MoE 层（Attention）复制，MoE 层分片存储；

②训练稳定性：Router z-loss、选择性精度（局部 float32）、小初始化方差（0.1VS1)、专家正则化；

③显存 / 算力：激活参数少但总参数需全加载（如 Mistral 8x7B 显存仍为 47B 级别）。

（5）代表模型：

①Switch Transformer 通过选择性精度、初始化改进和专家正则化，平衡训练稳定性与效率；

②GLaM 以 Decoder-only 稀疏 MoE 架构，实现 1.2 万亿参数高效激活，性能超越 GPT-3 且能耗大幅降低，凸显数据质量的关键作用；

③开源 MoE 模型中，Mistral 8x7B 和 DeepSeek-R1 凭借优化路由与架构，以低计算成本实现高性能；实践中可通过替换 FFN 层将 MoE 与 Llama2 结合。

3、【代码实践】：

以 GPT 和 Transformers 库为核心，通过调试剖析`model.generate()`的底层逻辑。

（1）Pipeline 接口分预处理、推理、后处理三步：

①预处理将文本转为`input_ids`和`attention_mask`；

②推理阶段`model.generate()`是核心，会合并生成配置（如`max_new_tokens`、`do_sample`）并选择解码策略；

③后处理把 token ids 还原为文本，区分返回完整 / 新增内容。

（2）解码流程中，`generate()`先确定生成模式（贪心 / 采样 / 束搜索等），再进入逐 token 循环：每轮通过 KV 缓存复用历史计算，模型输出 logits 后，经`LogitsProcessor`（温度缩放、TopK/TopP 过滤等）修正分布，再按策略选下一个 token（贪心选最大值、采样从分布随机抽），直到触发停止条件。

（3）关键调试技巧：从 Pipeline 入口断点切入，通过 “步入 + 转到定义” 回溯核心逻辑，关注变量流转；理解温度、TopK/TopP 等参数对生成随机性和多样性的调控，以及不同解码策略（贪心、采样、束搜索）的适用场景差异。

4、【上下文学习与提示词技术】

（1）概述：上下文学习室一种无需梯度反向传播或者权重更新，仅通过提示词或示例即可让大模型完成任务的交互范式，包括零样本、少样本（含单样本）两类模式；

（2）主要模式：

①零样本学习（Zero-Shot Learning）：仅提供任务指令 + 待处理输入，模型依赖预训练知识完成任务；实现的逻辑是预训练构建的共享语义空间（词汇表征 + 语义关联）+ 对齐微调后的指令遵循能力，引导输出 token 的概率分布；但也有局限性是复杂和定制化任务效果不稳定；

②少样本学习（Few-Shot Learning）：上下文补充规范的输入输出示例，明确演示预期效果；实现逻辑是上下文构建临时任务概率分布，注意力机制提炼并泛化示例中的模式；

（3）核心假设与底层原理：

①微观（感应头机制）：Transformer 的 “感应头” 实现 “匹配并复制”，回溯相似输入并预测对应输出，是少样本模式匹配的微观基础；

②中观（隐式学习动力学）：前向计算中，自注意力 + MLP 组合产生瞬时 “权重更新” 效应，通过激活影响输出分布（无持久化权重改动）；

③宏观（近似）：贝叶斯视角，把上下文学习视为对潜在任务的（近似）概率推断，是启发性解释框架，但未被严格证明等价；

（4）进阶提示词技术

①思维链（Chain-of-Thought, CoT）：引导模型显式输出中间推理步骤，拆解复杂问题，缓解 “直觉短路”；

②思维树（Tree of Thoughts, ToT）：将线性推理扩展为树状结构，通过 “生成候选 + 状态评估 + 选择与回溯” 实现多路径探索，避免局部最优。

③专用推理模型（如 OpenAI o1、DeepSeek-R1）：强化学习（提升逻辑推导 / 自我校验能力）+ 监督微调（稳定长序列推理输出），底层能力进化，将推理能力内化于权重，自动展开长推导过程。
