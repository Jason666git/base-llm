Seq2Seq 架构核心要点梳理
一、核心定位
Seq2Seq（序列到序列）是为解决多对多（不对齐） 序列转换任务设计的架构（如机器翻译、文本摘要），核心是编码器 - 解码器（Encoder-Decoder）框架，区别于传统 RNN/LSTM 无法处理 “输入输出序列长度不等、无严格对齐” 的问题。

策略	做法	优缺点
作为解码器初始状态	C 适配后作为解码器初始隐藏 / 细胞状态	直观，但长序列易信息稀释
作为解码器每步输入	C 与每步词嵌入拼接 / 相加后输入解码器	抗遗忘，但 C 是静态向量，无对齐

二、核心思想与架构

1. 对比自编码器
   自编码器：目标是重构输入（编码器压缩为潜在表示，解码器还原），用于降维 / 去噪；
   Seq2Seq：目标是转换序列（编码器压缩为语义表示，解码器生成全新目标序列），用于序列转换。
2. 整体架构（双组件）
3. 解码器关键机制
   初始化：上下文向量 C 初始化解码器初始状态；
   自回归生成：第一步输入sos，后续每步输入上一步生成的词元；
   因果约束：仅用单向 RNN，无法 “看到” 未来词元。
   三、关键实现细节
4. 词嵌入层
   独立嵌入：源 / 目标语言词汇表独立时（如英译中），编码器 / 解码器各用一套 Embedding；
   共享嵌入：词汇表重叠时（如文本摘要），共享 Embedding 减少参数、增强关联。

6. 特殊词元
   pad：填充对齐序列，计算损失时忽略；
   sos：解码器起始输入；eos：生成终止标志；unk：替换未登录词。
7. 损失计算
   核心：交叉熵损失，计算预测概率分布与真实词元的差异；
   处理：展平维度适配接口，ignore_index 忽略pad损失。
   四、训练与推理模式
8. 训练：教师强制（Teacher Forcing）
   做法：解码器每步输入用真实标签（而非自身预测），避免误差累积；
   优势：收敛快、稳定性高；Transformer 中可配合掩码实现时间步并行计算。
9. 推理：自回归生成
   基础策略：贪心搜索（选每步概率最高词元），简单但易局部最优；
   优化策略：束搜索（保留多个候选序列，选全局最优）；
   高效实现：缓存解码器每步状态，避免重复计算（仅用上一步词元和状态输入）。

12. 六、应用与局限
13. 应用场景
    跨模态转换：语音识别（音频→文本）、图像描述（图像→文本）、TTS（文本→音频）；
    NLP 任务：机器翻译、文本摘要、问答系统、生成式分类（统一任务范式）。
14. 核心局限：信息瓶颈
    问题：所有输入信息压缩到固定长度 C，长序列易丢失细节，解码器无法 “聚焦” 输入对应部分；
    解决方案：注意力机制（允许解码器动态关注输入序列不同部分），催生 Transformer 模型。
16. 七、关键结论
17. Seq2Seq 核心是 “先编码整段输入为语义向量，再解码生成输出”；
    教师强制加速训练，自回归 + 束搜索优化推理；
    信息瓶颈是核心缺陷，注意力机制是其关键改进方向；
    本质是 “编码器 + 条件语言模型”，是 Transformer 等现代生成模型的基础。
